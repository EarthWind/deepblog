# 动态内存与链式结构

前面学的数组大多是在编译时就确定大小，但真实程序往往需要“运行时再决定要多少空间”。这就是动态内存的用武之地。学完这一阶段，你会真正理解堆内存、`malloc`、`free`，并开始接触链表这种典型链式结构。

## 本阶段目标

- 理解栈、堆、静态区的基本区别。
- 掌握 `malloc`、`calloc`、`realloc`、`free`。
- 理解内存泄漏、悬空指针和重复释放。
- 能实现最基础的单链表节点操作。

## 核心知识

### 1. 为什么需要动态内存

如果数组长度在程序运行前无法确定，固定数组就不够灵活。例如：

- 用户输入要存多少个整数
- 文件中到底有多少条记录
- 链表节点数量会不断变化

这时就需要在运行时申请内存。

### 2. 常用函数

- `malloc(size)`：申请指定字节数
- `calloc(n, size)`：申请并清零
- `realloc(ptr, new_size)`：调整已有内存大小
- `free(ptr)`：释放内存

### 3. 三个高频问题

- 内存泄漏：申请了但没释放
- 悬空指针：释放后还继续用
- 重复释放：同一块内存释放两次

## 可运行示例：动态数组求平均值

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int n = 0;
    int i = 0;
    int sum = 0;
    int *nums = NULL;

    printf("How many numbers? ");
    scanf("%d", &n);

    nums = (int *)malloc(sizeof(int) * n);
    if (nums == NULL) {
        printf("malloc failed\n");
        return 1;
    }

    for (i = 0; i < n; i++) {
        printf("nums[%d] = ", i);
        scanf("%d", &nums[i]);
        sum += nums[i];
    }

    printf("average = %.2f\n", sum / (double)n);

    free(nums);
    nums = NULL;
    return 0;
}
```

## 示例说明

- `malloc` 返回的是一块堆内存的首地址。
- 使用后必须 `free`。
- `free` 后把指针置为 `NULL` 是一个好习惯，可以减少误用风险。

## 可运行示例：单链表头插法

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int value;
    struct Node *next;
} Node;

Node *create_node(int value) {
    Node *node = (Node *)malloc(sizeof(Node));
    if (node == NULL) {
        return NULL;
    }
    node->value = value;
    node->next = NULL;
    return node;
}

void print_list(Node *head) {
    Node *current = head;
    while (current != NULL) {
        printf("%d ", current->value);
        current = current->next;
    }
    printf("\n");
}

void free_list(Node *head) {
    Node *current = head;
    while (current != NULL) {
        Node *next = current->next;
        free(current);
        current = next;
    }
}

int main(void) {
    Node *head = NULL;
    int i = 0;

    for (i = 1; i <= 5; i++) {
        Node *node = create_node(i);
        if (node == NULL) {
            free_list(head);
            return 1;
        }
        node->next = head;
        head = node;
    }

    print_list(head);
    free_list(head);
    return 0;
}
```

## 为什么链表很重要

链表能帮你理解：

- 节点之间如何通过指针连接
- 为什么动态内存比固定数组更灵活
- 数据结构为什么依赖内存模型

## 常见误区

### 申请成功后不判空

虽然很多小程序里 `malloc` 通常成功，但规范代码必须检查返回值。

### `free` 后继续访问

```c
free(nums);
printf("%d\n", nums[0]);
```

这是未定义行为。

### 忘记释放链表所有节点

只 `free(head)` 并不够，后面的节点仍然泄漏。

## 学习建议

- 每申请一块内存，都要想清楚在哪里释放。
- 画图理解链表节点之间的连接关系。
- 先练单链表，再看双链表和循环链表。
