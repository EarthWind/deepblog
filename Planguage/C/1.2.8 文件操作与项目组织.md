# 文件操作与项目组织

前面写的大多数程序，一结束数据就没了。文件操作解决的是“把数据保存下来”这个问题，而项目组织解决的是“代码越来越多后怎样管理”这个问题。它们是从练习程序走向小型项目的关键一步。

## 本阶段目标

- 理解文件和流的基本概念。
- 掌握文本文件的打开、读写和关闭。
- 理解多文件组织的基本方式。
- 能写出一个最简单的持久化小程序。

## 核心知识

### 1. 文件与流

在 C 语言里，文件通常通过 `FILE *` 来操作。最基础的流程是：

1. 打开文件
2. 读或写
3. 关闭文件

### 2. 常用函数

- `fopen`
- `fclose`
- `fprintf`
- `fscanf`
- `fgets`
- `fputs`
- `fread`
- `fwrite`

初学阶段先把文本文件的函数用熟，再逐步看二进制读写。

### 3. 多文件组织

当代码变多时，通常会拆成：

- `.h`：声明
- `.c`：实现

例如：

- `student.h`
- `student.c`
- `main.c`

这是后续工程化的基础。

## 可运行示例：写入并读取文本文件

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    FILE *fp = fopen("student.txt", "w");
    char line[100];

    if (fp == NULL) {
        printf("open file for write failed\n");
        return 1;
    }

    fprintf(fp, "name=Alice\n");
    fprintf(fp, "score=96\n");
    fclose(fp);

    fp = fopen("student.txt", "r");
    if (fp == NULL) {
        printf("open file for read failed\n");
        return 1;
    }

    while (fgets(line, sizeof(line), fp) != NULL) {
        printf("%s", line);
    }

    fclose(fp);
    return 0;
}
```

## 示例说明

- `"w"` 表示写模式，文件不存在就创建，存在就清空重写。
- `"r"` 表示读模式，文件必须存在。
- `fgets` 适合按行读取文本。

## 再看一个更接近小项目的例子：保存成绩

```c
#include <stdio.h>

typedef struct {
    char name[20];
    int score;
} Student;

int main(void) {
    Student s = {"Tom", 88};
    FILE *fp = fopen("score.txt", "w");

    if (fp == NULL) {
        printf("failed to open file\n");
        return 1;
    }

    fprintf(fp, "name=%s\n", s.name);
    fprintf(fp, "score=%d\n", s.score);
    fclose(fp);

    printf("data saved to score.txt\n");
    return 0;
}
```

## 常见误区

### 打开文件后忘记关闭

这是非常常见的问题。无论读写是否成功，只要打开成功，就应在合适的位置关闭。

### 文件模式搞错

- `"w"`：写
- `"r"`：读
- `"a"`：追加

误用 `"w"` 会覆盖原文件内容。

### 不检查 `fopen` 返回值

文件路径错误、权限不足时，打开可能失败。必须判空。

## 学习建议

- 先练文本文件，再看二进制文件。
- 每次写文件代码都固定写上失败处理。
- 开始尝试把结构体、数组、文件操作组合起来做小项目。

## 本阶段练习

- 复制一个文本文件
- 统计文件中的行数
- 保存并读取学生成绩
- 把通讯录数据写入文件
- 拆分一个小项目为多个 `.c` 和 `.h` 文件

## 项目组织建议

一个简单项目可以这样组织：

```text
project/
  main.c
  student.h
  student.c
  file_util.c
```

你现在不必追求完整工程化，但要开始习惯：

- 把声明写进头文件
- 把实现放进源文件
- 让 `main` 只负责调度流程
