# DAOS 博客系列规划

这份目录用于沉淀一组基于 `daos` 仓库真实实现整理出来的博客选题，目标不是复述官方文档，而是给出一条更适合源码阅读者的、由浅入深的写作顺序。

## 文件说明

- `daos-blog-series.md`: 完整博客系列清单，按学习曲线拆成四个阶段，并为每篇文章给出写作重点、建议结构和关键代码入口。

## 推荐阅读顺序

1. 先理解 DAOS 的产品定位、控制面/数据面分层与仓库结构。
2. 再看 `daos_server`、`daos_engine`、`dmg`、`daos_agent` 这几个核心进程是如何协作的。
3. 然后进入 `pool`、`container`、`object`、`rebuild`、`rdb`、`rsvc` 等服务层。
4. 最后再深挖 `vos`、`bio`、`vea` 等底层存储实现，以及客户端 API、构建和测试体系。

## 设计原则

- 以源码实现为主线，优先引用 `src/` 下真实模块，而不是只讲概念。
- 以调用链为骨架，尽量回答“入口在哪、请求怎么走、状态存在哪、故障怎么处理”。
- 以层次递进为顺序，先建立整体认知，再进入模块细节和底层机制。
- 以写作者友好为目标，每篇都给出可直接展开成正式文章的提纲。
