# 标准库系统学习

标准库是 C 语言真正进入“可用状态”的关键。只会语法而不会用标准库，写程序会非常原始；但只会调用函数而不了解边界，也很容易出错。这一阶段的重点是把常用库按场景系统整理起来。

## 本阶段目标

- 熟悉最常用标准头文件的职责。
- 理解常见库函数的参数、返回值和边界条件。
- 知道什么时候该直接调用库函数，什么时候需要自己实现。

- `printf`

- `fputs`

### 2. 字符串与内存类

## 可运行示例：字符串与字符处理组合

```c
#include <stdio.h>
#include <string.h>
#include <ctype.h>

int main(void) {
    char text[100] = "DeepBlog C123";
    int i = 0;

    printf("length = %u\n", (unsigned)strlen(text));

    for (i = 0; text[i] != '\0'; i++) {
        if (isalpha((unsigned char)text[i])) {
            text[i] = (char)tolower((unsigned char)text[i]);
        }
    }

    printf("lower = %s\n", text);
    return 0;
}
```

这个例子把三个常见头文件连起来用了：

- `string.h`
- `ctype.h`
- `stdio.h`

## 可运行示例：动态内存与库函数协作

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int main(void) {
    char *buf = (char *)malloc(20);

    if (buf == NULL) {
        return 1;
    }

    memset(buf, 0, 20);
    strcpy(buf, "library");
    printf("buf = %s\n", buf);

    free(buf);
    return 0;
}
```

## 学习重点

### 1. 不只看“能不能用”，还要看“边界在哪”

例如 `strcpy` 能复制字符串，但如果目标空间不够，就会有风险。

### 2. 学会看返回值

很多函数通过返回值表达：

- 是否成功
- 结果是什么
- 错误发生在哪里

### 3. 优先用成熟库函数

初学阶段可以手写函数帮助理解，但真实项目中很多基础能力优先用标准库更可靠。

## 常见误区

### 把库函数当成黑盒

会用还不够，至少要知道：

- 输入是什么
- 输出是什么
- 会不会修改传入数据
- 会不会越界

### 看见函数名就猜功能

很多函数名字相似，例如 `strcpy`、`strncpy`，差别不能只靠感觉。

### 不查头文件就直接用

标准库函数通常需要对应头文件支持，忘记包含可能导致告警或错误。

## 学习建议

- 用“分类表”整理你最常用的函数。
- 每学一个库函数，就写一个最小示例。
- 把标准库当作工具箱，而不是知识点清单。
