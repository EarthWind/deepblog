# 入门与开发环境

第 1 阶段的目标不是“学会很多语法”，而是先把 C 语言的基本运行方式搞明白：代码写在哪里、怎样编译、怎样运行、程序入口在哪里、为什么一个 `.c` 文件能够变成一个可执行程序。

## 本阶段目标

- 理解 C 语言适合做什么。
- 能使用编译器把 `.c` 文件编译成可执行程序。
- 能看懂一个最小 C 程序的结构。
- 建立“先写最小程序，再逐步加功能”的习惯。

## 需要掌握的内容

### 1. C 语言是什么

C 语言是一门通用的过程式编程语言，特点是：

- 语法简洁，接近底层。
- 运行效率高。
- 可以直接操作内存。
- 是很多系统软件、嵌入式软件和基础库的核心语言。

适合学习和应用的方向包括：

- 数据结构与算法
- 操作系统基础
- 嵌入式开发
- 网络编程
- 高性能基础库

### 2. 开发环境的基本组成

想让一个 C 程序跑起来，通常需要这些工具：

- 编辑器：写代码
- 编译器：把代码翻译成机器能执行的程序
- 终端：执行编译命令和运行程序

常见编译器有：

- GCC
- Clang
- MSVC

### 3. 一个 C 程序的基本结构

一个最小 C 程序通常包含：

- 头文件引用
- `main` 函数
- 语句
- 返回值

`main` 是程序入口。操作系统启动程序时，会先执行 `main`。

## 第一个可运行示例

下面这段代码可以直接编译运行：

```c
#include <stdio.h>

int main(void) {
    printf("Hello, C!\n");
    printf("I am ready to learn C language.\n");
    return 0;
}
```

如果使用 GCC，可以这样编译：

```bash
gcc hello.c -o hello
```

运行后会输出：

```text
Hello, C!
I am ready to learn C language.
```

## 代码解析

- `#include <stdio.h>` 表示引入标准输入输出库，这样才能使用 `printf`。
- `int main(void)` 表示主函数没有参数，返回一个 `int`。
- `printf` 用来向控制台输出文本。
- `\n` 表示换行。
- `return 0;` 表示程序正常结束。

## 从源码到程序的大致过程

初学阶段不必深入细节，但要先知道有这条链路：

1. 写源文件，例如 `hello.c`
2. 编译器读取源码
3. 编译器生成目标文件和可执行程序
4. 操作系统加载可执行程序
5. 从 `main` 开始执行

先建立这个模型，后面学编译与链接时会更轻松。

## 常见问题

### 为什么必须写 `main`

因为对于普通 C 程序来说，`main` 就是默认入口。没有入口，程序就不知道从哪里开始执行。

### 为什么 `printf` 不能直接用

因为 `printf` 不是语言关键字，而是标准库函数。使用前要包含对应头文件。

### 为什么有时能省略 `return 0`

在现代 C 环境下，`main` 结束时通常会隐式返回 0，但学习阶段建议显式写出，逻辑更清晰。

## 常见误区

- 只会点 IDE 的运行按钮，不知道底层做了什么。
- 看见 `#include` 就背，不理解它是“把声明引进来”。
- 认为写代码就是写语法，忽略编译和运行过程。

```c

int main(void) {
    printf("Name: Deep Learner\n");
    printf("Goal: Master C language step by step.\n");
    printf("Next: Learn variables and control flow.\n");

```

这段程序依然非常简单，但已经体现了初学阶段最重要的一件事：先让程序稳定运行，再逐步增加复杂度。
