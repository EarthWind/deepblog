# 进阶阶段

当前面 8 个阶段完成后，就进入真正的 C 语言进阶部分。这个阶段不再只是“会写语法”，而是开始理解 C 语言背后的编译过程、内存机制、库函数边界、数据结构组织方式以及调试和代码质量方法。

## 使用方式

- 先看本页，明确进阶阶段的全局路线。
- 再按第 9 到第 13 阶段顺序逐篇阅读。
- 每篇中的示例代码都建议自己新建 `.c` 文件运行一次。
- 对“为什么会这样”比“怎么写”投入更多注意力。

## 建议节奏

- 第 1 周：完成预处理、编译与链接，建立对构建流程的整体理解。
- 第 2 到第 3 周：重点攻克类型、对齐、字节序、位运算和可变参数。
- 第 4 周：集中整理标准库，把高频函数按用途分类记忆。
- 第 5 到第 7 周：进入数据结构与算法实战，真正把 C 用起来。
- 第 8 周以后：加强调试、重构和代码质量控制，把已有项目重新打磨一遍。

## 学习目标

完成这一组文章后，你应当能够：

- 理解一个 C 程序从预处理到链接的大致流程。
- 理解更多与内存、类型、底层表示相关的细节问题。
- 正确使用常见标准库函数，并知道它们的边界。
- 用 C 实现基础数据结构和常见算法。
- 独立排查常见编译错误、运行时错误和逻辑错误。
