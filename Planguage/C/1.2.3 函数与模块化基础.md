# 函数与模块化基础

当一个程序不再只有十几行代码时，就必须学会“拆分”。函数就是 C 语言中最重要的拆分工具。你需要把一个完整问题切成多个小任务，每个函数只做一件相对明确的事情。

## 本阶段目标

- 理解为什么要使用函数。
- 掌握函数的声明、定义、调用和返回值。
- 理解局部变量、全局变量和作用域。
- 形成基本的模块化思维。

## 核心知识

### 1. 函数的价值

函数可以解决三个问题：

- 减少重复代码
- 提高可读性
- 让调试和修改更容易

例如“求最大值”这种逻辑，如果在多个地方都要用，就不应该写三遍。

### 2. 函数的基本组成

一个普通函数包含：

- 返回类型
- 函数名
- 参数列表
- 函数体

例如：

```c
int add(int a, int b) {
    return a + b;
}
```

### 3. 声明与定义

- 声明：告诉编译器“这个函数存在”
- 定义：真正写出函数实现

在函数调用发生在定义之前时，通常需要先声明。

### 4. 作用域

- 局部变量：只在当前代码块有效
- 全局变量：在整个文件甚至跨文件可见

学习初期要优先少用全局变量，因为它会增加耦合。

## 可运行示例：成绩统计函数版

```c
#include <stdio.h>

int max_of_three(int a, int b, int c) {
    int max = a;

    if (b > max) {
        max = b;
    }
    if (c > max) {
        max = c;
    }

    return max;
}

double average_of_three(int a, int b, int c) {
    return (a + b + c) / 3.0;
}

int main(void) {
    int a = 0;
    int b = 0;
    int c = 0;

    printf("Input three scores: ");
    scanf("%d %d %d", &a, &b, &c);

    printf("max = %d\n", max_of_three(a, b, c));
    printf("average = %.2f\n", average_of_three(a, b, c));

    return 0;
}
```

## 示例说明

- `max_of_three` 只负责比较大小。
- `average_of_three` 只负责求平均值。
- `main` 负责输入和调用。

这就是最基础的模块化：每个函数职责明确。

## 递归入门示例：阶乘

```c
#include <stdio.h>

int factorial(int n) {
    if (n <= 1) {
        return 1;
    }
    return n * factorial(n - 1);
}

int main(void) {
    int n = 5;
    printf("%d! = %d\n", n, factorial(n));
    return 0;
}
```

递归的关键是两点：

- 有终止条件
- 每次调用都更接近终止条件

## 常见误区

### 把所有逻辑都塞进 `main`

这是最常见的问题。短程序还勉强能看，稍微一复杂就会失控。

### 函数职责不清

一个函数同时做输入、计算、打印、写文件，后期会很难维护。

### 返回值类型不匹配

例如函数声明返回 `int`，实际却想表达浮点结果，会导致精度损失。

## 学习建议

- 每个函数尽量只做一件事。
- 函数名要表达意图，例如 `print_menu`、`calc_sum`。
- 先写声明，再写实现，帮助你形成接口意识。
