# 深入理解类型、内存与底层机制

这一阶段开始真正进入 C 语言“底层味道”最浓的部分。你会发现很多看起来只是语法细节的问题，背后其实都和内存布局、类型解释方式、机器表示有关。

## 本阶段目标

- 理解类型不仅是语法标签，也是内存解释规则。
- 理解 `sizeof`、对齐、字节序等底层现象。
- 能看懂基础位运算和可变参数用法。
- 建立更扎实的“内存模型”意识。

## 核心知识

### 1. 类型决定了什么

类型至少决定两件事：

- 占用多少内存
- 这段内存如何被解释

例如同样 4 个字节：

- 可能是一个 `int`
- 也可能是一段字符数据

### 2. `sizeof` 与 `strlen`

这两个名字很容易混，但意义完全不同：

- `sizeof`：类型或对象所占字节数
- `strlen`：字符串有效字符长度，不包括 `\0`

### 3. 对齐与填充

结构体成员并不一定紧挨着排布，编译器可能为了访问效率插入填充字节。

这也是为什么两个字段总大小相加，不一定等于结构体大小。

### 4. 字节序

多字节数据在内存中如何排列，常见有：

- 大端
- 小端

学习阶段先理解概念和检测方法即可。

### 5. 位运算

位运算适合做：

- 标志位控制
- 权限组合
- 掩码计算

## 可运行示例：比较 `sizeof` 和 `strlen`

```c
#include <stdio.h>
#include <string.h>

int main(void) {
    char text[] = "hello";

    printf("sizeof(text) = %u\n", (unsigned)sizeof(text));
    printf("strlen(text) = %u\n", (unsigned)strlen(text));

    return 0;
}
```

运行结果一般会体现出：

- `sizeof(text)` 是 6，因为包括结尾的 `\0`
- `strlen(text)` 是 5，因为只统计有效字符

## 可运行示例：位运算实现权限标记

```c
#include <stdio.h>

#define READ_FLAG  1
#define WRITE_FLAG 2
#define EXEC_FLAG  4

int main(void) {
    int permission = 0;

    permission |= READ_FLAG;
    permission |= WRITE_FLAG;

    printf("permission = %d\n", permission);

    if (permission & READ_FLAG) {
        printf("read enabled\n");
    }
    if (permission & WRITE_FLAG) {
        printf("write enabled\n");
    }
    if (!(permission & EXEC_FLAG)) {
        printf("exec disabled\n");
    }

    return 0;
}
```

这个例子能帮助你理解：

- `|` 用于设置标志
- `&` 用于检测标志
- `!` 可用于取反逻辑判断

## 可运行示例：结构体对齐观察

```c
#include <stdio.h>

struct Sample {
    char a;
    int b;
    char c;
};

int main(void) {
    printf("sizeof(struct Sample) = %u\n", (unsigned)sizeof(struct Sample));
    return 0;
}
```

在很多平台上，这个结果不会只是 `1 + 4 + 1 = 6`，而可能更大，这正是对齐带来的影响。

## 常见误区

### 认为变量类型只是为了让编译器检查语法

实际上类型还决定：

- 取值范围
- 运算规则
- 指针步长
- 内存解释方式

### 忽略整数溢出

整数超出范围后，结果可能和直觉不同。学习阶段至少要意识到它是真实存在的问题。

### 把 `void *` 当万能类型随便用

`void *` 只是“暂时不指定具体类型的指针”，使用前通常仍要转换成明确类型。
