# 指针专题

指针是 C 语言最核心、也最容易让人卡住的部分。真正理解指针之后，你会发现很多数组、字符串、函数参数、动态内存的问题，最终都可以回到“地址”和“内存”来解释。

## 本阶段目标

- 理解指针变量保存的是地址。
- 掌握取地址 `&` 和解引用 `*`。
- 理解指针与数组之间的关系。
- 能写出基础的指针参数代码。

## 核心知识

### 1. 指针是什么

指针变量存的不是普通值，而是某个对象在内存中的地址。

例如：

```c
int x = 10;
int *p = &x;
```

这里：

- `x` 是一个 `int`
- `&x` 是 `x` 的地址
- `p` 是一个指向 `int` 的指针

### 2. 解引用

如果 `p` 里存的是 `x` 的地址，那么：

```c
*p
```

表示“拿到 `p` 指向的那个整数值”。

所以：

```c
*p = 20;
```

其实就是把 `x` 改成 `20`。

### 3. 指针与数组

数组名在很多表达式里会退化为首元素地址。

例如：

```c
int a[3] = {10, 20, 30};
```

通常可以把 `a` 看作 `&a[0]`。

因此：

- `a[1]`
- `*(a + 1)`

在很多场景下表示的是同一个元素。

## 可运行示例：用指针交换两个变量

```c
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main(void) {
    int x = 10;
    int y = 20;

    printf("before: x = %d, y = %d\n", x, y);
    swap(&x, &y);
    printf("after : x = %d, y = %d\n", x, y);

    return 0;
}
```

## 为什么这个例子重要

如果不用指针，函数只能拿到参数的副本，无法修改原变量。使用指针后，函数拿到的是地址，就能通过地址找到原数据并修改。

## 可运行示例：指针遍历数组

```c
#include <stdio.h>

int main(void) {
    int nums[5] = {2, 4, 6, 8, 10};
    int *p = nums;
    int i = 0;

    for (i = 0; i < 5; i++) {
        printf("%d ", *(p + i));
    }
    printf("\n");

    return 0;
}
```

这段代码说明：

- `p` 指向数组首元素
- `p + i` 会根据元素类型自动移动
- `*(p + i)` 就是对应位置的元素值

## 常见误区

### 指针没有初始化就使用

```c
int *p;
*p = 10;
```

这是严重错误，因为 `p` 此时没有指向合法地址。

### 以为指针就是数组

数组和指针关系密切，但不是一回事：

- 数组是连续存储的一组元素
- 指针是保存地址的变量

### 看不懂声明

例如：

- `int *p[10]`：指针数组
- `int (*p)[10]`：数组指针
- `int **p`：二级指针

建议从“变量名和它旁边的运算符”开始往外读。

## 学习建议

- 学指针时一定要画图，把变量、地址、箭头画出来。
- 每写一段指针代码，都问自己：谁保存地址，谁被指向。
- 不要急着背复杂声明，先把最基础的 `int *p = &x;` 练到熟。

## 推荐小实验：观察地址变化

```c
#include <stdio.h>

int main(void) {
    int nums[3] = {10, 20, 30};
    int *p = nums;

    printf("p     = %p\n", (void *)p);
    printf("p + 1 = %p\n", (void *)(p + 1));
    printf("p + 2 = %p\n", (void *)(p + 2));

    return 0;
}
```

运行后你会发现地址是按 `int` 的大小往前跳的，这能帮助你真正理解“指针运算”。
