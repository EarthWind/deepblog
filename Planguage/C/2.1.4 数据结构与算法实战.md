# 数据结构与算法实战

到了这一阶段，C 语言就不只是“语言学习对象”，而是“解决问题的工具”。数据结构与算法是把前面学过的数组、指针、结构体、动态内存全部串起来的最佳场景。

## 本阶段目标

- 用 C 手写基础数据结构。
- 能实现并理解常见排序和查找算法。
- 理解“数据组织方式”如何影响程序性能和复杂度。

## 核心知识

### 1. 为什么数据结构重要

同样是存数据：

- 用数组和用链表，插入删除成本不同
- 用栈和用队列，访问顺序不同
- 用哈希表和顺序查找，查找效率不同

数据结构不是为了背定义，而是为了选择合适的组织方式。

### 2. 入门顺序建议

建议按下面顺序学：

1. 顺序表
2. 单链表
3. 栈
4. 队列
5. 哈希表基础
6. 树基础

### 3. 算法练习重点

初学阶段的重点不只是“记住代码”，而是理解：

- 输入输出是什么
- 什么时候结束
- 时间复杂度大概怎样
- 有没有边界情况

## 可运行示例：冒泡排序

```c
#include <stdio.h>

void bubble_sort(int arr[], int n) {
    int i = 0;
    int j = 0;

    for (i = 0; i < n - 1; i++) {
        for (j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

int main(void) {
    int arr[5] = {5, 2, 4, 1, 3};
    int i = 0;

    bubble_sort(arr, 5);

    for (i = 0; i < 5; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    return 0;
}
```

这个例子适合拿来练：

- 数组操作
- 循环边界
- 元素交换

## 可运行示例：二分查找

```c
#include <stdio.h>

int binary_search(int arr[], int n, int target) {
    int left = 0;
    int right = n - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;
}

int main(void) {
    int arr[6] = {1, 3, 5, 7, 9, 11};
    int index = binary_search(arr, 6, 7);

    printf("index = %d\n", index);
    return 0;
}
```

这个例子说明一个关键点：算法往往依赖前提条件。二分查找要求数组必须有序。

## 可运行示例：数组实现栈

```c
#include <stdio.h>

#define MAX_SIZE 5

typedef struct {
    int data[MAX_SIZE];
    int top;
} Stack;

void init_stack(Stack *s) {
    s->top = -1;
}

int push(Stack *s, int value) {
    if (s->top == MAX_SIZE - 1) {
        return 0;
    }
    s->data[++s->top] = value;
    return 1;
}

int pop(Stack *s, int *value) {
    if (s->top == -1) {
        return 0;
    }
    *value = s->data[s->top--];
    return 1;
}

int main(void) {
    Stack s;
    int value = 0;

    init_stack(&s);
    push(&s, 10);
    push(&s, 20);

    if (pop(&s, &value)) {
        printf("pop = %d\n", value);
    }

    return 0;
}
```

## 常见误区

### 只背代码，不理解结构意义

例如会写链表节点，但不知道为什么插入删除时要改指针。

### 忽略边界情况

排序空数组、查找不存在元素、栈满和栈空，这些都必须考虑。

### 以为算法题和工程无关

实际上很多真实业务都依赖合适的数据组织方式，只是形式不一定像题目。

## 学习建议

- 每学一个结构，都写“增删改查”最小版本。
- 每学一个算法，都先拿纸模拟一遍流程。
- 写完程序后，自己构造几组边界输入做测试。
