# Rust 由浅入深学习路线

> 目标：从零掌握 Rust，最终能够独立开发可靠的命令行工具、网络服务和系统级程序，并具备阅读优秀开源项目、定位性能问题与安全使用 `unsafe` 的能力。

## 一、学习原则

1. **先写再懂**：每学一个语法点，都用 10～30 行小程序验证。
2. **重视编译器提示**：Rust 编译错误通常包含修复方向，先完整阅读再搜索答案。
3. **优先掌握所有权**：所有权、借用和生命周期是后续集合、并发、异步与系统编程的基础。
4. **项目驱动**：每个阶段至少完成一个可运行、可测试、可说明设计取舍的项目。
5. **保持工程习惯**：持续使用 `cargo fmt`、`cargo clippy` 和 `cargo test`。

## 二、环境准备与工具链

### 学习内容

- 安装 Rust 工具链管理器 `rustup`
- 理解 stable、beta、nightly 工具链
- 熟悉 `rustc`、`cargo`、`rustfmt`、Clippy、rust-analyzer
- 掌握 Cargo 项目的基本目录结构
- 学会查看本地文档：`rustup doc`

### 必会命令

```bash
rustc --version
cargo --version
cargo new hello-rust
cargo run
cargo check
cargo build --release
cargo fmt
cargo clippy
cargo test
cargo doc --open
```

### 实践任务

- 创建并运行第一个 Cargo 项目
- 修改程序，使其读取命令行输入并输出问候语
- 分别比较 `cargo check`、`cargo build` 与 `cargo run` 的用途

### 验收标准

- 能独立创建、编译、运行和格式化 Rust 项目
- 能解释 debug 与 release 构建的区别

## 第二阶段：所有权、借用与切片

### 核心知识

- 栈与堆的基本区别
- 所有权三条规则
- 移动、复制与克隆
- 不可变引用与可变引用
- 借用规则和悬垂引用
- 字符串切片与数组切片
- `String` 与 `&str` 的区别和转换
- `Copy`、`Clone`、`Drop` 的基本意义

### 重点练习

- 分析每次赋值和函数调用是否发生 move
- 将接收 `String` 的函数改写为优先接收 `&str`
- 在不克隆数据的前提下返回字符串中的第一个单词
- 编写同时读、写集合的例子，理解借用作用域

### 阶段项目：文本统计器

读取文本文件，统计：

- 总字符数、单词数、行数
- 出现频率最高的若干单词
- 最长的一行

要求尽量使用引用，避免没有必要的 `clone()`。

### 验收标准

- 能准确解释 move、borrow、copy、clone 的区别
- 遇到借用检查错误时，能根据引用的作用域定位原因
- 能合理选择 `String`、`&String` 和 `&str`

### 实践任务

### 核心知识

- 项目：代码统计器、批量重命名器、增量备份工具

### 嵌入式与无标准库

- 项目：传感器采集、简单驱动、嵌入式状态机

### 游戏与图形

- ECS、渲染管线、输入系统、资源管理、数学基础

### 源码阅读方法

3. 顺着一条核心调用链阅读。
4. 记录模块边界、关键数据结构和错误模型。
5. 尝试修复小问题、补测试或完善文档。

## 四、贯穿全程的练习清单

- 每周至少阅读一次标准库文档
- 每周解决 3～5 道小练习，但避免只刷算法题
- 为每个阶段项目编写 README 和使用示例
- 记录典型编译错误及其根因
- 每两周重构一次旧项目，观察理解是否加深
- 从第五阶段开始，为新功能先写测试或至少同时补测试
- 从第六阶段开始，让所有项目通过格式化、Clippy 和测试检查

## 五、常见误区

- 为快速通过编译而到处添加 `clone()`
- 把所有字符串参数都写成 `&String`，忽略更通用的 `&str`
- 过早使用复杂生命周期或 `unsafe`
- 在业务库中大量使用 `unwrap()`
- 把 `Arc<Mutex<_>>` 当作所有并发问题的默认答案
- 在异步运行时中执行阻塞 I/O 或持有同步锁跨越 `.await`
- 未建立基准就开始“优化”
- 只看书和视频，不写可运行项目

## 六、毕业项目验收表

一个合格的综合项目应满足：

- [ ] 目录结构清晰，库与可执行入口职责分离
- [ ] 公共 API 有文档，README 包含运行方法与示例
- [ ] 错误类型明确，错误信息可定位问题
- [ ] 核心逻辑具有单元测试和集成测试
- [ ] 通过 `cargo fmt`、Clippy 和全部测试
- [ ] 没有无理由的 `clone()`、`unwrap()` 和大范围锁
- [ ] 异步任务具备超时、取消或并发限制
- [ ] `unsafe` 代码最小化并记录安全不变量
- [ ] 有性能基线或压力测试结果
- [ ] 有日志、配置、发布或部署说明
